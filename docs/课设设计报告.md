# 基于 RFID 与 MQTT 的物联网智能安防综合管控平台

## ——《射频识别原理与应用》《物联网信息安全技术》《物联网工程设计与实现》课程设计

## 一、作品概述

本作品设计并实现了一套完整的 RFID 物联网智能安防管控系统，综合运用射频识别、嵌入式开发、物联网通信、信息安全、Web 前后端等技术，构建了覆盖**感知层、网络层、应用层**的物联网三层架构。系统以 ESP32 为核心终端节点，集成 MFRC522 RFID 读写模块、多种环境传感器（红外火焰、MQ-2 烟雾、DHT22 温湿度、HC-SR501 人体红外），通过 MQTT 协议实现加密数据上传与远程指令下发，后端采用 Flask + SQLite 搭建数据网关与管理平台，前端使用 Vue3 + Element Plus + ECharts 提供可视化交互界面，并集成基于 Web Speech API + DeepSeek AI 的智能语音控制。

系统实现了 RFID 门禁刷卡认证、环境安全实时监测、远程控制与智能分析、用户角色权限管理、刷卡录入新卡、批量删除卡片等完整功能，并全面应用了 **AES-128 对称加密、SHA-256 哈希校验、session 认证、基于角色的访问控制（RBAC）** 等信息安全技术，满足物联网三层信息安全传输要求。

## 二、物联网三层架构

### 2.1 感知层（Perception Layer）

感知层采用 ESP32-WROOM-32E 单片机最小系统作为核心控制器，外接多种传感器和执行器，负责数据采集、状态感知与自动控制。

| 分类 | 器件型号 | 功能 | 接口 |
|------|---------|------|------|
| **RFID 模块** | MFRC522 | 13.56MHz 射频识别，读写 IC 卡 UID | SPI (SCK/GPIO18, MISO/GPIO19, MOSI/GPIO23, SS/GPIO5) |
| **人体红外** | HC-SR501 PIR | 检测人员靠近，触发刷卡流程 | GPIO27 (数字输入) |
| **火焰传感器** | 红外接收管模块 | 检测明火（760-1100nm 红外波长） | GPIO26 (数字输入, LOW=有火) |
| **烟雾传感器** | MQ-2 | 可燃气体/烟雾浓度检测 | GPIO34 (ADC1, 12位采样 0-4095) |
| **温湿度传感器** | DHT22 | 环境温度(°C)和相对湿度(%RH)采集 | GPIO13 (单总线) |
| **OLED 显示屏** | SSD1306 128×64 | 系统状态实时显示 | I2C (SDA/GPIO16, SCL/GPIO17) |
| **蜂鸣器** | 有源蜂鸣器 | 报警/提示音 | GPIO25 (数字输出) |
| **双色 LED** | 红/绿 LED | 状态指示（绿灯通过、红灯报警） | GPIO32/GPIO33 |
| **舵机** | SG90 | 电子门锁 (0°锁/90°开) | GPIO14 (LEDC PWM, 50Hz) |

**感知层软件（ESP32 C++ / Arduino 框架）：**

- 主循环频率 ~20Hz，轮询传感器状态
- 软件去抖处理（火焰需连续两次 LOW 确认，PIR 带 200ms 去抖）
- **自动控制逻辑**：检测到合法卡 → 舵机开锁 + 绿灯；检测到火焰 → 舵机开锁疏散 + 蜂鸣持续报警 + 红灯；检测到烟雾超标 → 开锁疏散 + 蜂鸣报警；人离开超时 → 自动锁门
- 环境监测 5 秒一次采样，MQ-2 具有 3700 阈值（可根据实际环境在 `env_sensor.h` 中校准）
- 数据通过 AES-128-CBC 加密后以 JSON 格式经 MQTT 上传

**感知层信息安全：** ESP32 端使用 mbedtls 库实现 AES-128-CBC 对称加密，密钥 `RFID2026SecretK1` (16 字节) 与固定 IV，所有 MQTT 负载均为十六进制密文传输，确保感知层数据不被明文窃听。

### 2.2 网络层（Network Layer）

**无线通信方案：**

ESP32 通过 WiFi (IEEE 802.11 b/g/n, 2.4GHz) 连接局域网路由器，基于 TCP/IP 协议栈与 Mosquitto MQTT Broker (端口 1884) 建立持久连接。

**MQTT 通信协议设计：**

| Topic | 方向 | 用途 | 安全 |
|-------|------|------|------|
| `rfid/system` | ESP32 → 服务器 | 心跳包 + 环境数据 (温度/湿度/烟雾/火焰/人体/门锁) | AES 加密 |
| `rfid/auth` | ESP32 → 服务器 | 身份认证结果 (刷卡通过) | AES 加密 |
| `rfid/alarm` | ESP32 → 服务器 | 报警信息 (非法卡/烟雾/火灾) | AES 加密 |
| `rfid/door` | ESP32 → 服务器 | 门状态变化 (开/关) | AES 加密 |
| `rfid/cmd` | 服务器 → ESP32 | 远程控制指令 + 白名单同步 | 明文(指令短,本地网络) |
| `rfid/users` | 服务器 → ESP32 | 完整白名单 JSON 下发 | 局域网内明文 |

数据格式采用 JSON，心跳包示例字段：
```json
{
  "uid": "", "username": "ESP32_GATE_001", "event": "heartbeat",
  "time": "2026-06-28T12:00:00", "temp": 24.5, "humidity": 62.1,
  "smoke": 2000, "fire": false, "door": false, "human": true
}
```

**网络层信息安全：** 
- AES-128-CBC 对称加密保护 MQTT Payload，密钥在 ESP32 固件 (`config.h`) 和 Python 服务端 (`config.py`) 保持一致
- SHA-256 哈希校验数据完整性，ESP32 端对每条消息计算哈希摘要
- 心跳包每 30 秒发送一次，含完整环境数据，支持离线检测 (<65 秒无心跳判定离线)
- PIR 人体状态变化即时触发 MQTT 上报 (`human_detected` / `human_left`)，不等待心跳周期

### 2.3 应用层（Application Layer）

#### 后端服务

基于 Python Flask 框架搭建，集成功能：

| 模块 | 文件 | 功能 |

| 分析引擎 | `analysis_engine.py` | 基于 GB/T 18883-2022、GB 50736-2012、GB 50325-2020 国标的环境数据智能解读 |

#### 前端界面 (Vue3 SPA)

| 实时监控 | 温湿度/烟雾/火焰环境数据、门锁状态、人体检测、实时日志流 |
| 远程控制 | 开门/锁门/测试报警按钮（管理员权限） |
| 数据分析 | 7 天趋势图、事件分布饼图、用户活跃度排行 |
| **智能分析** | 温度/湿度/烟雾 ECharts 实时曲线图，国标环境解读与建议 |
| **权限管理** | 用户管理（增删账户、角色分配）、RFID 卡管理（刷卡录入新卡、批量勾选删除） |
| 语音控制 | 右下角麦克风按钮，支持开门/锁门/查询温度/查询湿度/查询状态等自然语言指令 |

**应用层信息安全：**
- 用户密码采用 SHA-256 哈希存储，数据库不存明文
- 基于 session 的认证机制，`@login_required` 装饰器保护所有敏感路由
- 基于角色的访问控制 (RBAC)：admin 可远程开门、管理用户和卡片；user 只能查看
- 语音控制接口 `/api/smart_cmd`：查询类指令免登录，控制类指令（开门/锁门/报警）需要通过 login session 且角色为 admin
- API key 不入库（`config.py` 已加入 `.gitignore`）

## 三、RFID 子系统详细设计

### 3.1 读卡器选型

选用 **NXP MFRC522** 芯片模块，工作频率 13.56MHz，支持 ISO/IEC 14443 A/MIFARE 协议。

**选型依据：**
- 工作电压 3.3V，与 ESP32 GPIO 电平兼容，无需电平转换
- SPI 接口，通信速率最高 10Mbps，满足实时读卡需求
- 支持 RC522 内部天线驱动，无需外部功放即可实现 3-5cm 读卡距离
- 开源社区广泛支持（Arduino MFRC522 库），开发效率高
- 成本极低（~5 元），适合课程设计

### 3.2 电子标签

使用 **MIFARE Classic 1K (S50)** 非接触式 IC 卡：
- 13.56MHz 被动式 RFID 标签，读写器磁场供电
- 4 字节 UID 全球唯一标识符
- 1KB EEPROM 存储空间（16 扇区 × 4 块 × 16 字节）
- 三次认证机制保障数据安全

### 3.3 工作原理与通信协议

**ISO/IEC 14443-3 防冲突与选卡流程：**

```
REQA/WUPA (26h/52h) → ATQA 应答
ANTICOLLISION (93h 20h) → 级联1 UID
SELECT (93h 70h + UID + BCC) → SAK 应答
三级认证 (MIFARE Crypto1)
HALT (50h 00h) → 卡休眠
```

MFRC522 通过 SPI 与 ESP32 通信，MCU 发送命令字节，RC522 处理射频前端调制/解调 (ISO/IEC 14443-2 规定的 100% ASK 调制、Manchester 副载波编码)。

### 3.4 双重认证方案

系统实现了两级白名单认证机制：

1. **动态白名单**：服务器数据库维护授权卡列表，通过 MQTT (`rfid/users` topic) 同步到 ESP32 本地缓存 (最大 50 人)。管理员通过 Web 界面增删卡片时，数据库写入 + MQTT 推送秒级同步到 ESP32
2. **出厂默认回退**：4 张固定卡（张三/李四/王五/ZDW）写死在 ESP32 固件中。断网时自动回退，保证离线也能刷卡

## 四、信息安全技术综合应用

### 4.1 对称密码 (AES-128-CBC)

| 项目 | 参数 |
|------|------|
| 算法 | AES-128 |
| 模式 | CBC (Cipher Block Chaining) |
| 密钥 | `RFID2026SecretK1` (16 字节/128 位) |
| IV | `0123456789abcdef` (16 字节, 两端硬编码一致) |
| 填充 | PKCS7 |
| ESP32 实现 | mbedtls 库 (`src/crypto.cpp`) |
| Python 实现 | PyCryptodome 库 (`crypto_helper.py`) |

所有 MQTT 消息负载经 AES-128-CBC 加密后转为大写十六进制字符串传输，密钥仅在 ESP32 固件和服务器配置文件中保存，不通过网络传输。

### 4.2 哈希函数 (SHA-256)

- ESP32 端对每条 JSON 消息计算 SHA-256 哈希值，串口打印用于调试校验
- Python 端用户密码以 SHA-256 哈希值存储在 SQLite 数据库中，不保存明文
- 登录认证时对输入的密码计算哈希后与数据库对比，避免明文对比

### 4.3 数字签名 / HMAC

心跳包包含 SHA-256 校验和，接收端可验证消息完整性，防止数据在 MQTT 传输过程中被篡改。

### 4.4 身份认证与访问控制

| 层级 | 机制 | 说明 |
|------|------|------|
| 应用层 | Session 认证 | Flask session 机制，`@login_required` 装饰器保护 API |
| 应用层 | RBAC 角色控制 | admin/user 两级角色，`@admin_required` 保护管理接口 |
| 语音接口 | 分级控制 | 查询类免登录，控制类需 Session + admin 角色 |
| 密码存储 | SHA-256 哈希 | 不存明文 |

### 4.5 信息安全三层覆盖

| 层级 | 安全措施 | 对应技术 |
|------|---------|---------|
| **感知层** | AES-128-CBC 加密 | 对称密码 |
| **感知层** | 数据实时采集并加密上传，不存明文 | 数据保护 |
| **网络层** | MQTT Payload 十六进制密文 | 通信加密 |
| **网络层** | SHA-256 哈希校验完整性 | 哈希函数 |
| **应用层** | Session + RBAC | 身份认证 + 访问控制 |
| **应用层** | SHA-256 密码存储 | 口令保护 |
| **应用层** | 控制指令权限分级 | 授权管理 |

## 五、智能化功能

### 5.1 智能语音控制

- **本地规则模式（默认）**：基于 `difflib` 的模糊语义匹配，覆盖开/锁门/报警/查询等 20+ 种中文自然语言表达
- **AI 增强模式**：接入 DeepSeek API 实现不限表达方式的自然语言理解
- 通过 Web Speech API 在浏览器端采集语音，不依赖第三方语音服务

### 5.2 智能环境分析

基于国家标准对环境传感器数据进行智能解读：

| 国标 | 应用 |
|------|------|
| GB 50736-2012 《民用建筑供暖通风与空气调节设计规范》 | 温度舒适区间判定 (18-26°C) |
| GB/T 18883-2022 《室内空气质量标准》 | 湿度评估 (30%-60%)、烟雾评估 |
| GB 50325-2020 《民用建筑工程室内环境污染控制标准》 | 污染物阈值参考 |

分析引擎自动生成分级结论 (danger/warning/info/good)，每条结论附带详细解读、行动建议和国标引用。

### 5.3 刷卡录入（无键盘录入新卡）

管理员在网页点击"刷卡录入"→ 输入持卡人姓名 → ESP32 进入学习模式 → 把新卡放上读卡器 → UID 自动读取并经 AES 加密上报 → 服务端自动入库并同步白名单 → 录入完成。

### 5.4 环境联动自动控制

- **火焰检测** → 舵机自动开锁疏散 + 蜂鸣持续报警 + 红灯 + MQTT 紧急上报
- **烟雾超标** → 自动开锁疏散 + 蜂鸣报警（安全设计：火灾/烟雾时优先保障人员逃生）
- **合法刷卡** → 舵机开锁 + 绿灯 10 秒 → 超时自动锁门 + MQTT 上报 door_close
- **非法刷卡** → 立即锁门（防止尾随）+ 蜂鸣报警 + 红灯 + MQTT 上报 unauthorized
- **人离开** → 检测到无人 1.5 秒后退出刷卡模式，门自动关闭

### 5.5 非法闯入防范

非法卡刷卡或门未正常关闭时，系统自动锁门并触发声光报警，事件经 AES 加密上报至管理后台记录。

## 六、数据库设计

| 表名 | 用途 | 字段 |
|------|------|------|
| `accounts` | Web 登录账户 | id, username, password(SHA-256), role, created |
| `users` | RFID 卡白名单 | uid(PK), username, role |
| `logs` | 事件日志 | id, uid, username, event, access_time |

**event 事件类型：**
`login`(刷卡认证通过)、`door_open`(开门)、`door_close`(关门)、`unauthorized`(非法刷卡)、`heartbeat`(心跳)、`system_start`(启动)、`human_detected`(有人靠近-即时)、`human_left`(人离开-即时)、`card_learned`(刷卡录入)

> 心跳记录不写入数据库，仅更新内存缓存，避免长期运行导致日志表膨胀。系统运行 14 天仅产生 200+ 条有效事件记录。
